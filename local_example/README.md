# Local 2-Node Example

Simulates a 2-node distributed CNN inference deployment on a **single machine**, using two loopback addresses:

| Virtual node | IP        | Role                              |
|--------------|-----------|-----------------------------------|
| `node_A`     | 127.0.0.1 | Start node — runs VGG16 blocks 0–6  |
| `node_B`     | 127.0.0.2 | Second node — runs VGG16 blocks 7–15 |

The strategy is **pipelined model partitioning**: `node_A` processes the early feature-extraction layers; the intermediate output (~0.8 MB, the smallest point in VGG16) is transferred over loopback to `node_B`, which completes inference and returns the top-3 ImageNet predictions.

---

## Files in this directory

| File | Description |
|------|-------------|
| `setup_local.py` | Generates `cc_graph.gml`, `net_rules/`, and `hs.txt` for the 2-node setup |
| `cc_graph.gml` | The 2-node CC graph (generated) |
| `hs.txt` | The placement strategy — 16 lines, one per VGG16 block (generated) |
| `net_rules.tsv` | Delay lookup table for `core.py` — empty (no simulated delays over loopback) |
| `net_rules/` | Per-node TSV files (generated, not used at runtime) |
| `benchmarks/` | Benchmark data directory |
| `benchmarks/nodes/node_A.json` | PartiBench output for node_A (generated by Step 2) |
| `benchmarks/benchmark.json` | Merged benchmark (generated by Step 3) |

---

## Prerequisites

Create and activate a virtual environment with the required dependencies (see the top-level `README.md` for details):

```bash
python3 -m venv ../partibench_env
source ../partibench_env/bin/activate
pip install "numpy<1.24" mxnet gluoncv networkx psutil
```

All commands below assume this environment is active and are run from the `local_example/` directory.

---

## Steps

### Step 1 — Generate local configuration 

```bash
python setup_local.py
```

This creates `cc_graph.gml`, `net_rules/`, and `hs.txt`. 

---

### Step 2 — Run PartiBench 

Profiles all VGG16 blocks locally. Takes ~7 minutes on a typical machine.

```bash
cp net_rules/node_A_rules.tsv net_rules.tsv
python ../benchmarking/benchmark.py node_A vgg16 True
```

Output: `node_A.json`. 

---

### Step 3 — Merge benchmark data 

```bash
cp node_A.json benchmarks/nodes/node_A.json
python ../benchmarking/merge_bench.py
```

Output: `benchmarks/benchmark.json`. 

---

### Step 4 — Start node_B's deployment agent

Open a **new terminal**, activate the environment, and run:

```bash
cd local_example
source ../partibench_env/bin/activate
python ../orchestrator/local_deploy.py --ip 127.0.0.2
```

You should see:
```
Waiting for component deployment info...
```

Leave this terminal open.

---

### Step 5 — Start node_A's deployment agent

Open another **new terminal**, activate the environment, and run:

```bash
cd local_example
source ../partibench_env/bin/activate
python ../orchestrator/local_deploy.py --ip 127.0.0.1
```

You should see:
```
Waiting for component deployment info...
```

Leave this terminal open.

---

### Step 6 — Run the orchestrator

In a **third terminal** (or the original one), run:

```bash
cd local_example
source ../partibench_env/bin/activate
python ../orchestrator/place.py \
    --start-node node_A \
    --graph cc_graph.gml \
    --bench benchmarks/benchmark.json \
    --hs hs.txt
```

This sends component assignments to both node agents. You'll see the 16 strategy entries printed, then the orchestrator exits.

---

### Step 7 — Trigger inference

Switch to **node_A's terminal** (Step 5). After receiving its components it will prompt:

```
Enter 's' to start inference or 'r' to prepare for re-deployment:
```

Type `s` and press Enter. The PRODUCER sends `data/ship.png` through the pipeline. After a few seconds you'll see the end-to-end time and top-3 predictions:

```
PRODUCER: Inference took: XXXX ms
PRODUCER: Results:
container ship, container vessel: 84.32%
wreck: 5.17%
lifeboat: 3.91%
```

The system then repeats automatically every 5 seconds for 15 more runs.

## Troubleshooting

**`ConnectionRefusedError` when running place.py** — one or both `local_deploy.py` instances are not running. Ensure Steps 4 and 5 are active before running Step 6.

**`Address already in use` on port 6999** — a previous `local_deploy.py` is still running. Find and kill it: `pkill -f local_deploy.py`

**`FileNotFoundError: net_rules.tsv`** — `core.py` reads this file at import time from the current working directory. Make sure you run all commands from `local_example/`.

**VGG16 weights download** — the first run downloads ~500 MB of VGG16 weights into `vgg16_data/`. Requires internet access.
